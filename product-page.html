Step 1

Install

How this works. You host your data in Power BI on your own Microsoft tenant. The MCP server runs locally on your machine and queries Power BI through Microsoft's API using your existing account. The AI copilot (GitHub Copilot, Claude, Cursor) processes data locally. Nothing leaves your environment.

Option A: One-liner install (recommended)

Open a terminal and run:

terminal
curl -sL proxuma.io/install | bash
terminal
curl -sL proxuma.io/install | bash
PowerShell
irm proxuma.io/install.ps1 | iex

The installer automatically:

  1. Checks Python, VS Code, and Git — if anything is missing, it opens the download page and waits for you to install it
  2. Clones the repository and installs Python packages — the MCP framework, Azure authentication, PII detection (Presidio + spaCy)
  3. Runs the auto-discovery wizard — opens your browser for Microsoft sign-in, finds your workspaces and datasets, and writes the config automatically
  4. Configures VS Code — registers the MCP server, enables Agent mode, copies prompt files, installs GitHub Copilot
  5. Opens VS Code — with the project folder ready to go
Everything stays local. The installer only downloads open-source Python packages from PyPI. No data is sent anywhere. The MCP server runs on your machine and connects to Power BI through Microsoft's official API using your own credentials.

Step 2

Connect

The installer's setup wizard handles authentication and configuration automatically. Here's what happens:

2a. Microsoft sign-in

  1. A browser window opens automatically with the Microsoft login page
  2. Sign in with the Microsoft account that has your Power BI license
  3. You may see a permissions consent screen — click "Accept". The server requests read-only access to Power BI data and Fabric schema definitions.
  4. The browser shows "Authentication complete" — you can close it

2b. Pick your workspace and dataset

After sign-in, the wizard lists your available workspaces and datasets. Pick the ones you want to use as defaults — no need to hunt for GUIDs in browser URLs.

The config is written automatically to ~/.powerbi-mcp/config.json%USERPROFILE%\.powerbi-mcp\config.json:

config.json
{ "default_workspace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890", "default_dataset_id": "f9e8d7c6-b5a4-3210-fedc-ba0987654321", "workspaces": { "My Production Workspace": "a1b2c3d4-e5f6-7890-abcd-ef1234567890" }, "datasets": { "My Dataset": "f9e8d7c6-b5a4-3210-fedc-ba0987654321" } }
After first login, you won't need to log in again. Tokens refresh automatically. You'll only need to re-authenticate if you delete the auth cache, change your password, or are inactive for 90+ days.

2c. Verify in VS Code

  1. Open the project folder in VS Code (the installer may have done this already)
  2. Open the Command Palette: Cmd+Shift+PCtrl+Shift+P
  3. Search for "MCP: List Servers"
  4. You should see powerbi listed. If not, see Troubleshooting.

Then check that prompt files are loaded:

  1. Open Copilot Chat: Cmd+Shift+ICtrl+Shift+I
  2. Switch to Agent mode (click the mode dropdown at the top of the chat — it must say "Agent", not "Ask" or "Edit")
  3. Type # and check that you see #powerbireport and #powerbidashboard
Agent mode is required. MCP tools and prompt files only work in Agent mode. If you don't see the Power BI tools, switch from "Ask" to "Agent" in the Copilot Chat mode selector.

Test the connection with a simple request:

Copilot Chat (Agent mode)
You: List my Power BI workspaces

Copilot should call the list_workspaces tool and return something like:

expected response
Available workspaces: - My Production Workspace ID: a1b2c3d4-e5f6-7890-abcd-ef1234567890 - Development ID: 11223344-5566-7788-99aa-bbccddeeff00
If no tools appear in Copilot: Make sure you're in Agent mode (not Ask mode). Check that .vscode/mcp.json exists. Reload VS Code. See Troubleshooting for more.

Step 3

Generate Your First Report

Now the real thing. In Copilot Chat (Agent mode), type:

Copilot Chat
You: #powerbireport what is my monthly revenue trend?

The #powerbireport prefix loads the report generator prompt. The AI will then:

  1. Search your data model — calls search_schema with keywords like "revenue", "monthly"
  2. List available measures — calls list_measures to see what's already built
  3. Run DAX queries — executes 4-8 targeted queries to collect KPIs, breakdowns, and trends
  4. Generate the HTML — writes a complete, self-contained report with data tables, analysis, and actionable findings

The output is a single HTML file you can open in a browser. It includes all CSS and JavaScript inline — no external dependencies.

Tips for good prompts

  • Be specific: "What is my SLA compliance rate by client?" works better than "show me SLA data"
  • Ask business questions: "Which clients are most profitable?" not "select all from billing table"
  • One topic per report: Don't combine "revenue and tickets and SLA" in one prompt

Dashboard builder

The dashboard prompt works the same way:

Copilot Chat
You: #powerbidashboard create a resource utilization dashboard

This generates an interactive mockup with page tabs, sample data, and step-by-step Power BI Desktop build instructions. Written for people who want a head start, not a textbook.

Reference

MCP Tools Reference

The MCP server exposes 7 tools. You typically don't call them directly — the prompt files handle that. But it helps to know what's available.

ToolDescriptionWhen to use
list_workspaces List all workspaces you have access to First time setup, finding your workspace ID
list_datasets List datasets in a workspace Finding your dataset ID, checking what's available
execute_dax Run a DAX query and return results as JSON Any data retrieval. The workhorse tool.
search_schema Search for measures, columns, and tables by keyword Finding relevant columns before writing DAX. Use this instead of get_schema.
list_measures List all measure names (without definitions) Quick overview of what's in the data model
list_fabric_items List items in a Fabric workspace Seeing reports, lakehouses, and other Fabric items
get_schema Full semantic model definition Avoid. Returns 10MB+ on large models. Use search_schema instead.
Never use get_schema on large models. It returns the full TMDL definition as decoded base64, often exceeding 10MB. This will overwhelm most AI context windows and may crash the session. Use search_schema with specific keywords instead.

Default IDs

If you've set default_workspace_id and default_dataset_id in config.json, you can omit these from any tool call. The server falls back to your defaults automatically.

Troubleshooting

setup.sh says Python 3.10+ is required, but I have Python installed

The script checks for python3.12, python3.11, python3.10, and python3 in that order. If none of these return version 3.10+, it fails.

Check: python3 --versionpython --version. If below 3.10, install a newer version: brew install python@3.12sudo apt install python3.12download from python.org.

Browser doesn't open for Azure login

The server uses InteractiveBrowserCredential which opens your default browser. If it doesn't open:

  • You may be in a headless environment (SSH, Docker) — use --device-code flag instead
  • Try clearing the cached auth: rm ~/.powerbi-mcp/auth_record.jsondel "%USERPROFILE%\.powerbi-mcp\auth_record.json"
  • Restart VS Code and try again
"AADSTS65001: The user or administrator has not consented"

Your Azure AD tenant requires admin consent for public client flows. Options:

  1. Ask your Azure AD admin to grant consent for the public client
  2. Register a custom Azure AD app (Azure Portal → App registrations → New → Public client → add Power BI delegated permissions → grant admin consent)
VS Code Copilot doesn't show the Power BI tools
  • Make sure .vscode/mcp.json exists and the Python path is absolute
  • Switch to Agent mode (not Ask mode) in the Copilot Chat mode dropdown
  • Reload VS Code: Cmd+Shift+PCtrl+Shift+P → "Developer: Reload Window"
  • Check the Output panel (MCP channel) for error messages
Typing #powerbireport doesn't do anything
  • Check that .github/prompts/powerbireport.prompt.md exists
  • Enable in VS Code settings: "chat.promptFiles": true
  • You must be in Agent mode, not Ask mode
  • Reload VS Code after adding prompt files
AI generates a short summary instead of a full report

This happens with smaller models (GPT-4o mini, Haiku). Fixes:

  • Switch to a more capable model (GPT-4o, Claude Sonnet/Opus) in your AI settings
  • Reply with: That's a summary. Generate the FULL HTML report with all sections, all table rows, and all CSS. Minimum 800 lines.
  • Break into steps: first "run the DAX queries", then "generate the HTML from those results"
DAX query returns "HTTP 400 Bad Request"

Common causes:

  • Column or table name doesn't exist — use search_schema to verify
  • Missing quotes around names with spaces: 'Table Name'[Column Name]
  • Incorrect DAX syntax — test the query in Power BI Desktop first
HTTP 403 Forbidden

Your account doesn't have access to this workspace or dataset. Verify:

  • You can see the dataset in Power BI Service
  • You have at least Viewer role on the workspace
  • The dataset allows XMLA endpoint connections (check workspace settings)
Using with Claude Desktop or Cursor instead of VS Code

Claude Desktop: Add to ~/Library/Application Support/Claude/claude_desktop_config.json~/.config/Claude/claude_desktop_config.json%APPDATA%\Claude\claude_desktop_config.json:

{ "mcpServers": { "powerbi": { "command": "/path/to/python3C:\\path\\to\\python.exe", "args": ["/path/to/powerbi-copilot/server/server.pyC:\\path\\to\\powerbi-copilot\\server\\server.py"] } } }

Cursor: Settings → MCP → Add Server → name: powerbi, command: your python3python.exe path, args: server.py path.

Enterprise Deployment

For IT teams rolling this out across multiple machines. All examples skip the interactive wizard and pre-configure everything.

Silent install via MDM

terminal
curl -sL proxuma.io/install | bash -s -- \ --workspace-id "a1b2c3d4-e5f6-7890-abcd-ef1234567890" \ --dataset-id "f9e8d7c6-b5a4-3210-fedc-ba0987654321" \ --silent
PowerShell (Intune)
.\setup.ps1 ` -WorkspaceId "a1b2c3d4-e5f6-7890-abcd-ef1234567890" ` -DatasetId "f9e8d7c6-b5a4-3210-fedc-ba0987654321" ` -Silent

Config from URL

Host a shared config.json on your internal network and have the installer pull it automatically:

terminal
./setup.sh --config-url https://it.acme.com/powerbi-mcp-config
PowerShell
.\setup.ps1 -ConfigUrl "https://it.acme.com/powerbi-mcp-config"

Environment variables

As an alternative to config.json, the server reads these environment variables at startup:

VariableDescription
POWERBI_MCP_WORKSPACE_IDDefault workspace GUID — overrides config.json
POWERBI_MCP_DATASET_IDDefault dataset GUID — overrides config.json
POWERBI_MCP_CONFIGPath to an alternative config.json file

Device code flow (SSH / headless)

For environments without a browser (SSH sessions, Docker containers, CI runners), use the device code flow:

terminal
./setup.sh --device-code
PowerShell
.\setup.ps1 -DeviceCode

The installer prints a URL and a one-time code. Open the URL on any device (phone, another computer), enter the code, and sign in with your Microsoft account. Authentication completes on the headless machine.

Combine flags freely. For example: --device-code --workspace-id XXX --dataset-id YYY skips both the browser login and the interactive workspace picker.
