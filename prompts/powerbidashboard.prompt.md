# Power BI Dashboard Builder

Generate a complete HTML file with an interactive dashboard mockup AND step-by-step Power BI build instructions.

**Dashboard Topic:** $ARGUMENTS

**If no topic is specified, ask the user what dashboard they want to build.**

---

## CRITICAL OUTPUT RULES

**You MUST follow these rules. Violations make the output useless.**

1. **OUTPUT = complete, self-contained HTML file.** Not a summary. Not an outline. The FULL file with working tab navigation, CSS charts, and copy-paste DAX measures.
2. **Do NOT summarize.** Write every DAX measure, every build step, every conditional formatting rule. If there are 20 measures, write all 20.
3. **Do NOT skip the build instructions.** The instructions section is typically LONGER than the mockup section. Every visual gets step-by-step clicks.
4. **Minimum output: 800 lines of HTML.** A proper dashboard builder is 1500-3000 lines. If your output is under 800 lines, you've summarized.
5. **Use REAL table/column names** from the data model. Never guess column names.

---

### Data Anonymization

All data from MCP tools is pre-anonymized. Client names appear as Client_A/B/C, resource names as Resource_1/2/3, etc. Use these aliases as-is. The user will deanonymize the output locally after generation.

---

## Workflow

### Phase 1: Explore the Data Model

Extract 3-5 keywords from the topic and search for relevant tables, columns, and measures.

**If the user has set default IDs in `config.json`, you can omit workspace_id/dataset_id.**

```
search_schema(search_term="<keyword>")
list_measures()
```

Then pull sample data with DAX to understand what's in the tables:
```
execute_dax(dax_query="EVALUATE TOPN(5, 'TableName')")
```

**NEVER use `get_schema`** — it returns >10MB and will crash.

### Phase 2: Design the Dashboard

Plan 4-6 dashboard pages:
- **Executive overview** with KPI cards
- **Breakdown/matrix** with conditional formatting
- **Trend analysis** (time series)
- **Detail/drill-through** page
- **Distribution** (who/what is over/under)

For each page, define: visuals, exact field mappings, color rules, slicers.

### Phase 3: Write DAX Measures

Create 15-25 DAX measures. Each must:
- Reference actual table and column names from the data model
- Be ordered: independent measures first (1-8), dependent measures after
- Include a "Copy DAX" button

### Phase 4: Generate the HTML File

**OUTPUT: A single, complete HTML file with TWO main sections:**

## Part 1: Dashboard Preview

Visual mockup of each dashboard page with:
- Page tab navigation (click to switch pages)
- KPI cards with realistic sample data
- CSS-only bar charts, doughnut charts, or heatmaps
- Tables with conditional formatting (color-coded cells)
- Slicer placeholders

Use CSS grid/flexbox for layout. No JavaScript charting libraries needed — CSS backgrounds and gradients work fine for mockups.

## Part 2: DAX Measures

List every measure with:
- Measure number and name
- Which table to create it in (right-click → New measure)
- The full DAX formula in a `<pre><code>` block
- A "Copy DAX" button
- Brief explanation of what it calculates

**OUTPUT: Write ALL measures. If there are 20, write 20. Do NOT say "measures 8-20 follow the same pattern."**

## Part 3: Build Instructions

**Written for someone who has NEVER built a Power BI dashboard.**

Start with "Step 0: Before You Build Anything":
- How to open the dataset
- Why measures come first (before visuals)
- How to add a visual (click icon → drag fields)
- How to add pages (+ button, double-click to rename)
- How to apply conditional formatting (paint roller → Cell elements → Background color)

Then page-by-page instructions:
```
Page 1: [Name]
1. In the Visualizations pane, click the Matrix icon (looks like a grid)
2. Drag `column_name` into the Rows well
3. Drag `[Measure Name]` into the Values well
4. Click the paint roller icon (Format)
5. Expand "Cell elements"
6. Toggle "Background color" to ON
7. Click the fx button
8. Set Format by: Rules
9. Add rule: If value >= 90, color #059669 (green)
...
```

**OUTPUT: Write instructions for EVERY visual on EVERY page. Do NOT summarize or skip pages.**
